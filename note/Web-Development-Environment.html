Web 开发环境搭建

Node.js 环境

采用 fnm，nvs，nvm 等 Node.js 版本管理器安装 Node。

  1. 采用 Node Version Manager nvm 安装 Node，采用 NPM Registry Manager nrm 管理 npm 源，安装 Yarn 1

如采用安装版 nvm，采用默认安装路径 C:\Program Files\nodejs；

# 建议配置镜像源
$ nvm node_mirror https://npmmirror.com/mirrors/node/
$ nvm npm_mirror https://npmmirror.com/mirrors/

# 查看最新 LTS 版本
$ nvm list available
# 安装和使用一个版本
$ nvm install <version>
$ nvm use <version>

检查：$ node -v ,$ npm -v, $ yarn -v

  1. 配置 npm 全局模块存放路径、cache 的路径
$ cd \pro*\nod*
$ md node_global node_cache
$ npm config set prefix "C:\Program Files\nodejs\node_global"
$ npm config set cache "C:\Program Files\nodejs\node_cache"
  1. 设定 Node 环境变量

新建 NODE_PATH: C:\Program Files\nodejs\node_global\node_modules

Path 追加 C:\Program Files\nodejs\node_global\

  1. 设定 Yarn 环境变量

%localappdata%\Yarn\bin

%localappdata%\Yarn\Data\global\node_modules\.bin

  1. 视网络情况，更换 npm 源
# 方式一，通过 nrm 管理模块镜像源
$ yarn global add nrm
# 看看有哪些 npm 镜像源
$ nrm ls
# 使用一个速度快的
$ nrm use <registry>

# 方式二，手动配置模块镜像源
npm set registry https://registry.npmmirror.com
npm config get registry
npm cache clean -f

yarn config set registry https://registry.npmmirror.com

# 其它一些二进制包
npm config set disturl https://npmmirror.com/mirrors/node
npm config set sass_binary_site https://npmmirror.com/mirrors/node-sass
npm config set phantomjs_cdnurl https://npmmirror.com/mirrors/phantomjs
npm config set chromedriver_cdnurl https://npmmirror.com/mirrors/chromedriver
npm config set operadriver_cdnurl https://npmmirror.com/mirrors/operadriver
npm config set electron_mirror https://npmmirror.com/mirrors/electron
npm config set python_mirror https://npmmirror.com/mirrors/python
npm config set electron_builder_binaries_mirror https://npmmirror.com/mirrors/electron-builder-binaries
npm config set node_sqlite3_binary_host_mirror https://npmmirror.com/mirrors/sqlite3
npm config set puppeteer_download_host https://npmmirror.com/mirrors
npm config set sharp_binary_host https://npmmirror.com/mirrors/sharp
npm config set sharp_libvips_binary_host https://npmmirror.com/mirrors/sharp-libvips
npm config set canvas_binary_host_mirror https://npmmirror.com/mirrors/canvas
  1. 全局安装常用 npm 包 (可选择)

$ yarn global add http-server browser-sync terminal-translate eslint gulp-cli grunt-cli npm-check-updates

启动前端工程

在前端项目 package.json 文件所在目录命令行 (CLI) 执行 yarn install 或 npm install 安装依赖包即可，HG/Git 版本控制工具中添加忽略目录 node_modules。

安装完依赖后，查看 package.json 文件中 scripts 属性，鼠标放在属性名上，会弹出启动命令的提示，按实际情况启动项目。

开发工具

  • 配置能识别 0oO1lI 的字体
  • 编辑器：Visual Studio Code
  • or Visual Studio IDE Community 版
  • or WebStorm

先配置一个适合编码的字体，如 "editor.fontFamily": "consolas, 'Source Code Pro', 'Lucida Console', 'Ubuntu Mono', '等线', '微软雅黑 Light', '思源黑体 CN Light'",；

计算机配置较好，想折腾，推荐免费的编辑器 Visual Studio Code 并安装扩展调教，
计算机配置一般，或不想折腾，推荐 IDE，独占所有系统资源；

  • 不想折腾编辑器配置，从公共分享的配置下载配置：Visual Studio Code 安装 Settings Sync 同步扩展，可在设置中设置"sync.gist": "5f0eb9482b96be16eb6865cbc7728d29"项，然后 Ctrl+Shift+P 中执行Sync: Advanced Options → Sync: Download Settings from Public GIST，然后执行 Sync: Download Settings 同步，视网络情况需要一段时间。
  • IDE，直接选择免费的 Visual Studio IDE Community 版，安装时候选择“单个组件”勾选安装适合 Web 开发的组件。

可同步编辑器的设置和快捷键，方便在不同计算机上快速搭建开发工具。

开发环境

Web 开发环境只是整个开发环境中的一环，建议安装和配置较完整的开发/应用环境，如 Windows Platforms 的 .NET Runtime、Microsoft Visual C++ Redistributable、Microsoft Edge WebView Runtime，Java Platforms 的 Java Development Kit，以及 Python、Ruby、ImageMagick 等基础环境。
